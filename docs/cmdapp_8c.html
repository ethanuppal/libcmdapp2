The behavior parameter is easily the most confusing. I have written a comprehensive breakdown of the parameter.

Parameters

long_optThe long version of the option.
behaviorCharacteristics of the option, such as whether it takes an argument.
resultA pointer to a string initialized with the argument passed to this option.
descriptionA description of the option.
Returns
A handle to whether the flag was passed, or NULL on failure.

◆ ca_parse()

int ca_parse ( void * user_data)

Runs the parser on the command line arguments.

After this function returns, all arguments will be initialized with proper values. At each stage during the parsing, the user-provided callbacks will be invoked with user_data as its parameter (see ca_set_callbacks()).

Precondition
ca_init() must have been called.
Returns
Zero on success, nonzero on failure.
Time Complexity
This function runs in O(nm) time where n is the number of parsed options and arguments and m is the number of options, assuming that each long option is roughly constant time in comparison. In other words, if options a, b, and c all support multiflag, then -abc would correspond with n=3.

◆ ca_synopsis()

void ca_synopsis ( const char * synopsis)

Registers a synopsis.

A synopsis is a textual description of how the command should be run. Note that whatever interpretations or scheme you intend for your synopsis is entirely up to you. If NULL is passed, this function has no effect.

Example
Here are some example synopses with possible interpretations.
  • "subcommand [OPTION]..." means that the program can take a subcommand followed by a series of options.
  • "[OPTION]... FILE" means that the program can take a series of options followed by a filename.
